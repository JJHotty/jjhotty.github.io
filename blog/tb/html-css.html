Inline vs Inline Block...

Research the difference between display inline vs. inline blocks.

In html the code is built up by different 'tags'. A tag is just a indicator that tells the browser what will be displayed.. ie <p> <p/> tags tell the browser that you are inputting a paragraph. And similarly <strong> <strong/> tags tell the browser to make the content between the tags bold.
To zoom out a little bit further- HTML also has tags dedicated to laying items out within a page.. Ways to keep certain content on the left-hand side of the screen, or things at the top, or even things that overlay over the top of everything else.

To start my research off with I wanted to find out what the display property was-- And here is what I found..
There are a few common display tag options. ie. None; which stops a segment of code from showing up at all. Flex; which converts the items into flexbox items which are great for laying items around the screen. Block; which tell the browser that this item needs it's own line and needs to extend from the far left to the far right. Grid; and so on..

The next portion was to figure out- What makes something a block element, and what makes something an inline element.
From what I could find- Block elements will always tell the browser to lay them across the whole display. They will extend from the far left to the far right-- By default*. They can always be edited using block properties such as height, width, padding and margin.
An inline item though- Only takes up as much space as the item within it. So if I was to place an inline tag within a paragraph like a <strong> <strong/> tag. The paragraph of text within the tags would be edited and adjusted into bold text. And the text within those tags wouldn't demand to be placed within it's own line. But it's important to understand that Inline elements cannot be edited like a block element. You cannot edit it's height, width, or padding or margin.
Now to the final item.. Inline blocks are a hybrid. They inherit their size from the content within them like an inline item. But can be edited and adjusted like a block element- meaning you can adjust their margin, padding, height and width like a block.

To Summarise. Both Block elements and Inline Block elements are ideal for their use-cases. Sometimes you need an item to act in a way that inherits their size, and other times you want to be able to specify their size, or they will act in a familiar way- By acting like a typical block element.
I really appreciated the opportunity to research the topic between block and inline-block elements and I now understand way more about how certain blocks work within the html system.

The next time I create a blog post I intend to add an inline-block element just to play around with the concept. Thanks for reading.
